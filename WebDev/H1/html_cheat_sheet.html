HTML Cheatsheet

a - Hyperlink

A hyperlink consists of 2 basic elements, a string of text and another resource (a document or webpage for instance). It allows users to click it and access the linked resource.

Example

Mozilla Developer Network 

  <!-- HTML -->
  <a href="https://developer.mozilla.org/">  
      Mozilla Developer Network
  </a>

p - Paragraph

Paragraphs are elements that contain text and are separated by whitespaces when used in succession.

Example

Lorem ipsum dolor sit amet consectetur, adipisicing elit. Repellendus sequi voluptatibus magnam hic odio nisi rem consectetur libero nobis voluptate quibusdam ipsum repudiandae reiciendis harum, nam ducimus temporibus dignissimos facilis! 


  <!-- HTML -->
  <p>
    Lorem ipsum dolor sit amet consectetur, adipisicing elit. Repellendus sequi  
    voluptatibus magnam hic odio nisi rem consectetur libero nobis voluptate  
    quibusdam ipsum repudiandae reiciendis harum, nam ducimus temporibus dignissimos  
    facilis!
  </p>

h1, h2, ... , h6 - Section Headings

Example

This is a level 4 heading


  <!-- HTML -->
  <h4>
    This is a level 4 heading  
  </h4>

div - Content Division

Creates a "division", which can be understood as container, often relevant for styling.

Example

Help me! I'm trapped inside a div. 

  <!-- HTML -->
  <div>
      Help me! I'm trapped inside a div.  
  </div>

span - Content Span

Spans are inline elements that allow you to group and style content.

Example

Hello, I will use a span to add style.


  <!-- HTML -->
  <p>
    Hello, I will use a <span style="color: #E28964">span</span> to add style.  
  </p>

strong - Strong Importance

The visual effect of the strong tag is to make the text bold. However, it is used semantically to indicate that the bolded text has strong importance.

Example

Important: Tags can have semantic meaning. 

  <!-- HTML -->
  <strong>Important</strong>: Tags can have semantic meaning.

i - Idiomatic Text

The i tag will italicize text. However, it is used semantically to convey that text should be read with a different tone. When trying to emphasize with italics the em tag should be used.

Example

Its Leviosa, not Leviosaaa 

  <!-- HTML -->
  Its <i>Leviosa</i>, not <i>Leviosaaa</i>

u - Unarticulated Annotation/Underline

This tag will underline text. It is used semantically for annotations like annotate spelling errors.

Example

I lov syntax highlighters! 

  <!-- HTML -->
  I <u style="text-decoration:#f00 wavy underline">lov</u> syntax highlighters!

b - Bring attention to/Boldface

This tag, as mentioned previously, makes the text bold; but without additional importance. Meaning, it makes the text stand out but only stylistically (not semanitcally).

Example

Poor b tag, it was left behind by the semantic tags. 

  <!-- HTML -->
  Poor b tag, it was left behind by the <b>semantic</b> tags.

strike - Strikethrough

Strike is deprecated in HTML5 and it was used to cross out (strikethrough) text. It has been replaced by the semantic del and s tags.

Example

Research about the strike tag 

  <!-- HTML -->
  <strike>Research about the strike tag</strike>

small - Side comment

Makes text smaller and is mainly used to add side-comments or small print (like legal information).

Example

Some license information 

  <!-- HTML -->
    <small>Some license information</small>

em - Emphasis

Used semantically to emphasize text with italics.

Example

I need more time! 

  <!-- HTML -->
  I <em>need</em> more time!

header - Header

Used to add introductory content like titles and authors for articles.

Example

Real fake doors

Come get fake doors 

  <!-- HTML -->
  <header>
    <h4>Real fake doors</h4>  

    Come get fake doors
  </header>

section - Generic Section

Sections are divisions (divs) with additional semantic meaning. A section is considered to be standalone and are usually expected to have headings.

Example

Section Example

This is a very important section! 

  <!-- HTML -->
  <section>
    <h4>Section Example</h4>
    
    This is a very important section!  
  </section>

article - Article Contents

Articles are used for blog entries, news articles, product cards or in general self-contained content.

Example

Chat GPT is scary

Chat GPT, Open AI's chatbot is likely going to affect many jobs


  <!-- HTML -->
  <article>
    <h4>Chat GPT is scary</h4>
    
    <p>Chat GPT, Open AI's chatbot is likely going to affect many jobs</p>  
  </article>

aside - Aside

Used semantically to create a section that is indirectly (tangentially) related to the main content of the page. Note that additional styling is often used to indicate that an aside tag is being used.

Example

Linear Algebra is a field of mathematics that has important applications in Computer Science.
Vectors and matrices are used a lot in game development! 

  <!-- HTML -->
  Linear Algebra is a field of mathematics that has important applications in Computer  
  Science.
  <aside>
    Vectors and matrices are used a lot in game development!. 
  </aside>

footer - Footer

A footer is a sort of subsection that contains information about the section where it is contained such as copyright and authors. Websites often have a footer for the main content, but sections inside a document can also have footers themselves.

Example

Made with ❤️ by me. 

  <!-- HTML -->
  <footer>
    Made with ❤️ by me. 
  </footer>

nav - Navigation Section

The nav section is used to provide links that allow users to navigate a webpage (a document) or website (multiple documents).

Example

  • Back to Coursework

  <!-- HTML -->
  <nav>
    <ul>
      <li>
        <a href="joaquinbadillo.github.io/A01026364_TC2005B">Back to Coursework</a>  
      </li>
    </ul>  
  </nav>

img - Image Embed

The img tag allows you to add images. It uses attributes to specify the path of the image or relevant information like a description in case the image cannot be shown or needs to be read aloud. It cannot contain children as it is a void element and therefore has no ending tag.

Example


  <!-- HTML -->
  <a href="https://github.com/JoaquinBadillo">
    <img src="https://img.shields.io/github/followers/JoaquinBadillo?style=social"   
      alt="GitHub Followers Shield">
  </a>

br - Line Break

It is a void element (it has no ending tag and thus cannot contain children) that creates a line break.

Example

Text before line break
Text after line break 

  <!-- HTML -->
  Text before line break  
  <br>
  Text after line break
Note: Sometimes the self-closing tag <br/> is used for readability or compatibility, but HTML ignores the slash, as self-closing tags do not exist in HTML.

hr - Horizontal Rule/Thematic Break

Creates a horizontal rule (a horizontal line) that semantically indicates a theme break.

Example

Front-end developers design User Interfaces with a User Experience in mind.
Back-end developers implement business logic and server-side functionality. 

  <!-- HTML -->
  Front-end developers design User Interfaces with a User Experience in mind.  
  <hr>
  Back-end developers implement business logic and server-side functionality.

form - Form

A form can be understood as a section that contains fields for the users to fill and submit.

Example

Consumer Satisfaction Survey

Did you understand the purpose of a form?
Yes No 

  <!-- HTML -->
  <form>
    <h4>Consumer Satisfaction Survey</h4>
    Did you understand the purpose of a form?
    <br/>

    <input type="radio" name="Yes" value="yes"/> Yes  
    <input type="radio" name="No" value="no"/> No
    <br/>

    <button type="submit">Submit!</button>
  </form>

table - Table

Create a HTML table: a bidimensional set of data.

Example

Actual Positive Actual Negative
Predicted Positive True Positive False Positive
Predicted Negative False Negative True Negative 

  <!-- HTML -->
  <table border="1px solid white">  
    <tr>
      <td></td>
      <th>Actual Positive</th>
      <th>Actual Negative</th>
    </tr>
  
    <tr>
      <th>Predicted Positive</th>
      <td>True Positive</td>
      <td>False Positive</td>
    </tr>
  
    <tr>
      <th>Predicted Negative</th>
      <td>False Negative</td>
      <td>True Negative</td>
    </tr>
  </table>

ul - Unordered List

Contains an unordered list of items (a list without indexing, like bullet point lists).

Example

TODO List

  • Finish HTML Cheatsheet
  • Practice LeetCode
  • Sleep

  <!-- HTML -->
  <h4>TODO List</h4>
  <ul>
    <li><del>Finish HTML Cheatsheet</del></li>  
    <li>Practice LeetCode</li>
    <li>Sleep</li>
  </ul>

ol - Ordered List

Represents an ordered list of items, by default these are numbered lists, but they can also use other kinds of labels like letters or roman numerals.

Example

Sorting Algorithm

  1. If not sorted:
    1. Sort left half
    2. Sort right half
    3. Merge sorted halves
  2. Else:
    1. Return sorted array

  <!-- HTML -->
  <h4>Sorting Algorithm</h4>
  <ol>
    <li>If size > 1:</li>
        <li>Sort left half</li>
        <li>Sort right half</li>
        <li>Merge sorted halves</li>  
      </ol>
    <li>Else:</li>
      <ol type="a">
        <li>Return sorted array</li>
      </ol>
  </ol>
