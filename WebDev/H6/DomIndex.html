  1. ¿Qué es el DOM?
    2. El DOM es un modelo de objetos que representa la estructura de un documento HTML o XML. El DOM se utiliza para acceder y manipular los elementos de un documento HTML o XML.
    El DOM (Document Object Model) es una interfaz de programación de aplicaciones (API) para documentos HTML y XML. Esencialmente, el DOM es una representación en memoria del contenido de un documento web, como un árbol de objetos, que permite a los programadores acceder, manipular y actualizar dinámicamente el contenido de una página web.
    El DOM es esencial para la creación de aplicaciones web dinámicas e interactivas. Al manipular los elementos y atributos del DOM, los programadores pueden crear efectos visuales, cambiar el contenido de la página en tiempo real, validar formularios y realizar otras operaciones avanzadas en la interfaz de usuario.
    En resumen, el DOM es la estructura que permite a los programadores acceder y manipular dinámicamente el contenido de una página web, lo que lo convierte en una herramienta esencial para el desarrollo de aplicaciones web modernas y dinámicas.
    Verificado según MDN (DOM)
  2. ¿Qué es un nodo?
    3. Un nodo es un elemento de un documento HTML. Los nodos pueden ser etiquetas HTML, texto, comentarios, etc.
    En el contexto del Document Object Model (DOM), un nodo es un objeto que representa un elemento, atributo, texto o cualquier otro tipo de contenido dentro de un documento HTML o XML.
    Cada nodo en el DOM es considerado como un objeto con propiedades y métodos que pueden ser accedidos y manipulados por el programador mediante el lenguaje de programación utilizado en la página web (como JavaScript). Cada nodo tiene un conjunto de características únicas, como su nombre, valor, tipo, posición en la jerarquía del árbol DOM, y así sucesivamente.
    Los nodos pueden tener hijos y padres, lo que significa que pueden ser anidados dentro de otros nodos para formar una estructura jerárquica. Por ejemplo, el nodo de un elemento HTML <body> podría tener nodos hijos como <p>, <div>, <img>, etc.
    En resumen, un nodo es un objeto en el DOM que representa un elemento, atributo o cualquier otro contenido dentro de un documento HTML o XML, y que puede ser accedido y manipulado por el programador mediante el uso de un lenguaje de programación.
    Verificado según MDN (Node - DOM)
  3. ¿Cómo se accede a un elemento del DOM?
    4. Para acceder a un elemento del DOM en HTML, se puede utilizar JavaScript y las funciones que proporciona para manipular el contenido de la página web. Algunas funciones que se pueden utilizar para acceder a elementos del DOM son getElementById(), getElementsByClassName() y querySelectorAll().
    Para acceder a un elemento del DOM por su ID en JavaScript, se puede utilizar el método getElementById() y pasar el ID del elemento como argumento: 
    
<script>
    // Obtener el elemento con ID "mi-elemento"
    const elemento = document.getElementById("mi-elemento");
    // Hacer algo con el elemento, como cambiar su contenido    
    elemento.innerHTML = "Nuevo contenido";
</script>
    También es posible acceder a los elementos por su clase utilizando el método getElementsByClassName() o querySelectorAll(). Estos métodos devuelven una lista de elementos que tienen la clase especificada. Aquí hay un ejemplo que utiliza querySelectorAll() para obtener una lista de elementos con la clase "mi-clase": 
    
<script>
    // Obtener todos los elementos con la clase "mi-clase"
    const elementos = document.querySelectorAll(".mi-clase");
    
    // Iterar sobre la lista de elementos y hacer algo con cada uno    
    elementos.forEach(function(elemento) {
        elemento.innerHTML = "Nuevo contenido";
    });
</script>
    En resumen, para acceder a un elemento del DOM en HTML, se puede utilizar JavaScript y los métodos getElementById(), getElementsByClassName() o querySelectorAll().
    Verificado según MDN (Selectors) y MDN (Document API)

  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?
    6. getElementby* es una función que se utiliza para obtener un elemento del DOM, y se puede utilizar de diferentes maneras:
    • getElementById: Obtiene un elemento por su id
    • getElementsByClassName: Obtiene un elemento por su clase
    • getElementsByName: Obtiene un elemento por su nombre
    • getElementsByTagName: Obtiene un elemento por su etiqueta
    querySelector* es una función que se utiliza para obtener un elemento del DOM, y se puede utilizar de diferentes maneras:
    • querySelector: Obtiene un elemento por su selector
    • querySelectorAll: Obtiene todos los elementos por su selector
    Ambas, getElementBy* y querySelector*, son funciones de JavaScript que se utilizan para acceder a elementos del DOM. La principal diferencia entre ellas es el método que utilizan para acceder a los elementos.
    getElementBy* es un conjunto de funciones que se utilizan para acceder a elementos del DOM utilizando su nombre de etiqueta, ID o clase. Estas funciones devuelven un único elemento que coincide con el criterio de búsqueda especificado. Por ejemplo, getElementById se utiliza para acceder a un elemento por su ID, mientras que getElementsByClassName se utiliza para acceder a elementos por su clase. Estas funciones devuelven un objeto de elemento DOM que se puede utilizar para manipular el contenido o estilo del elemento.
    Por otro lado, querySelector* es una función que se utiliza para acceder a elementos del DOM utilizando un selector CSS. Esta función devuelve el primer elemento que coincide con el selector CSS especificado. Por ejemplo, querySelectorAll se utiliza para acceder a todos los elementos que coinciden con el selector CSS especificado. Esta función devuelve una lista de elementos DOM que se pueden utilizar para manipular el contenido o estilo de los elementos.
    En resumen, la principal diferencia entre getElementBy* y querySelector* es que la primera se utiliza para acceder a elementos del DOM por nombre de etiqueta, ID o clase, mientras que la segunda se utiliza para acceder a elementos del DOM utilizando un selector CSS.
    Verificado según MDN (Selectors) y MDN (Document API)
  6. ¿Qué es un evento?
    7. Un evento es una acción que ocurre en la página web que puede ser detectada por el navegador. Algunos ejemplos de eventos son: hacer click en un botón, mover el mouse, escribir en un campo de texto, etc.
    Para detectar un evento, se debe utilizar la función addEventListener sobre el elemento HTML que se quiere detectar el evento. Esta función recibe dos parámetros: el nombre del evento y una función que se ejecutará cuando ocurra el evento.
    Por ejemplo, para detectar el evento click sobre el botón Enviar, se debe escribir el siguiente código: 
    
    const boton = document.querySelector('#enviar');
    boton.addEventListener('click', function() {
        console.log('Se hizo click en el botón');
    });
    En este caso, la función que se ejecutará cuando se haga click en el botón es una función anónima. También se puede pasar una función con nombre como parámetro: 
    
    const boton = document.querySelector('#enviar');
    boton.addEventListener('click', mostrarMensaje);

    function mostrarMensaje() {
        console.log('Se hizo click en el botón');
    }
    En HTML, un evento es cualquier acción que se produce en un elemento de la página web, como hacer clic en un botón, mover el cursor sobre una imagen, presionar una tecla, etc. En JavaScript, los eventos se utilizan para controlar la interacción del usuario con la página web y para ejecutar código en respuesta a una acción específica del usuario.
    Los eventos en HTML se definen como atributos en las etiquetas HTML, como por ejemplo: 
    
<button onclick="alert('Hola mundo!')" >
    Haz clic aquí
</button>
    En este ejemplo, el evento onclick se define en el botón para ejecutar una función alert que muestra un mensaje cuando el usuario hace clic en el botón.
    En JavaScript, los eventos se pueden manipular utilizando la propiedad on de un elemento del DOM. Por ejemplo, para agregar un evento click a un botón en JavaScript, se puede utilizar el siguiente código: 
    
<button id="mi-boton">
    Haz clic aquí
</button>

<script>
    // Obtener el elemento con ID "mi-boton"
    const boton = document.getElementById("mi-boton");
    
    // Agregar un evento "click" al botón
    boton.onclick = function() {
        alert("Hola mundo");
    };
</script>
    En resumen, los eventos son acciones que ocurren en un elemento de una página web y se utilizan para controlar la interacción del usuario y ejecutar código en respuesta a una acción específica del usuario. En HTML, los eventos se definen como atributos en las etiquetas HTML, mientras que en JavaScript, los eventos se pueden manipular utilizando la propiedad on de un elemento del DOM.
    Verificado según W3schools (JS Events)
  7. ¿Qué es un listener?
    8. En JavaScript, un listener es una función que se ejecuta cuando se produce un evento en un elemento de la página web. Los listeners se utilizan para controlar la interacción del usuario con la página web y para ejecutar código en respuesta a una acción específica del usuario. 
    
<script>
    // Obtener el elemento con ID "mi-boton"
    const boton = document.getElementById("mi-boton");

    // Agregar un listener al botón
    boton.addEventListener("click", function() {
        alert("El botón ha sido presionado");
    });
</script>
    En JavaScript, un listener es una función que se asocia a un elemento del DOM y se activa cuando se produce un evento en ese elemento. Los listeners se utilizan para controlar la interacción del usuario con la página web y para ejecutar código en respuesta a una acción específica del usuario.
    Un listener se define utilizando el método addEventListener en JavaScript. Por ejemplo, para agregar un listener click a un botón en JavaScript, se puede utilizar el siguiente código: 
    
<button id="mi-boton">
    Haz clic aquí
</button>
<script>
    // Obtener el botón por su ID
    const boton = document.getElementById("mi-boton");

    // Agregar un evento "click" al botón
    boton.addEventListener("click", function() {
        alert("Hola mundo!");
    });
</script>
    En este ejemplo, la función anónima se ejecuta cuando se produce un evento click en el botón.
    La ventaja de utilizar listeners en lugar de definir eventos en las etiquetas HTML o en la propiedad on del elemento del DOM es que se pueden agregar varios listeners para el mismo evento en el mismo elemento sin sobrescribirse. Además, los listeners se pueden eliminar utilizando el método removeEventListener en JavaScript.
    En resumen, en JavaScript, un listener es una función que se asocia a un elemento del DOM y se activa cuando se produce un evento en ese elemento. Los listeners se definen utilizando el método addEventListener en JavaScript y se utilizan para controlar la interacción del usuario con la página web y ejecutar código en respuesta a una acción específica del usuario.
    Verificado según Web Dev Simplified (Event Listeners) y MDN (Events)
  8. ¿Qué es un callback?
    9. En JavaScript, callback es una función que se pasa a otra función como un argumento. Esta función se invoca, después, dentro de la función externa para completar alguna acción.
    Los callbacks se suelen utilizar para ejecutar código en respuesta a una acción específica del usuario.
    En JavaScript, un callback es una función que se pasa como argumento a otra función y se ejecuta después de que se completa la operación de la función original. El callback se utiliza para manejar la respuesta de la función original y realizar una acción adicional en respuesta a esa respuesta.
    Los callbacks se utilizan comúnmente en JavaScript para manejar eventos asíncronos, como la carga de recursos externos (imágenes, archivos CSS o JavaScript, etc.), solicitudes de servidor y animaciones. En lugar de esperar a que se complete la operación antes de continuar con el resto del código, se utiliza un callback para manejar la respuesta cuando se reciba, permitiendo que el código continúe ejecutándose mientras se espera la respuesta.
    Un ejemplo de un callback en JavaScript es el siguiente: 
    
function miFuncion(callback) {
    // Realizar alguna operación aquí
    
    // Llamar al callback después de que se completa la operación
    callback();
}
    
// Llamar a miFuncion y pasar una función como callback
miFuncion(function() {
console.log("La operación ha sido completada");
});
    En este ejemplo, la función miFuncion acepta un argumento callback, que es una función que se ejecutará después de que se completa la operación de la función. Luego, se llama a miFuncion y se le pasa una función anónima como callback, que simplemente imprime un mensaje en la consola.
    En resumen, en JavaScript, un callback es una función que se pasa como argumento a otra función y se ejecuta después de que se completa la operación de la función original. Los callbacks se utilizan comúnmente para manejar eventos asíncronos y permitir que el código continúe ejecutándose mientras se espera la respuesta.
    Verificado según MDN (Callback Functions)
  9. ¿Qué es un objeto de evento?
    10. En JavaScript, un objeto de evento es un objeto que contiene información sobre un evento que se ha producido en un elemento de la página web. Los objetos de evento se utilizan para controlar la interacción del usuario con la página web y para ejecutar código en respuesta a una acción específica del usuario.
    Para crear un objeto de evento, se puede usar cualquier objeto de evento de JavaScript nativo (MouseEvent, KeyboardEvent, FocusEvent, etc.) o se puede usar la interfaz Event personalizada
    En JavaScript, un objeto de evento es un objeto que contiene información sobre un evento que ha ocurrido en el navegador. Cuando un evento ocurre, el navegador crea un objeto de evento que se puede utilizar para obtener información sobre el evento y manejarlo de manera programática.
    Los objetos de evento se utilizan comúnmente con listeners de eventos, que son funciones que se ejecutan en respuesta a un evento específico. Cuando se ejecuta un listener de evento, el objeto de evento se pasa como argumento a la función.
    El objeto de evento contiene información sobre el tipo de evento, el elemento del DOM que desencadenó el evento, la posición del mouse (en el caso de eventos de mouse), las teclas del teclado que se han presionado (en el caso de eventos de teclado), y otras propiedades que dependen del tipo de evento.
    Por ejemplo, el siguiente código muestra cómo se puede utilizar un objeto de evento para detectar cuando se hace clic en un botón y obtener información sobre el evento: 
    
const boton = document.getElementById('mi-boton');

boton.addEventListener('click', function(evento) {
    // Prevenir el comportamiento predeterminado del botón
    evento.preventDefault();

    // Obtener información sobre el evento
    console.log('Tipo de evento:', evento.type);
    console.log('Elemento que desencadenó el evento:', evento.target);
    console.log('Coordenadas del mouse:', evento.clientX, evento.clientY);
});
    En este ejemplo, se agrega un listener de evento click al botón con el ID mi-boton. La función que se ejecuta cuando se hace clic en el botón recibe el objeto de evento como argumento y utiliza sus propiedades para obtener información sobre el evento.
    En resumen, en JavaScript, un objeto de evento es un objeto que contiene información sobre un evento que ha ocurrido en el navegador. Se utiliza comúnmente con listeners de eventos para manejar eventos de manera programática y obtener información sobre el evento que ha ocurrido.
    Verificado según MDN (Event Interface) y MDN (Events Intro)

Conclusiones

Experiencia general

Creo que las inteligencias artificiales son una buena herramienta para realizar consultas rápidas, si y solo si ya se tiene un conocimiento previo del tema. Siempre sentí que había un riesgo de que las cosas que sugerían o que afirmaban podían ser incorrectas.

También parecen ser buenas herramientas cuando se les da algo ya existente: utilizar copilot para comentar fue muy eficiente y en este caso sin fallas, mientras que GPT usaba ejemplos todo el tiempo (los cuales probablemente pudo consultar en su entrenamiento).

¿Cuándo y para qué usarías ChatGPT y cuándo Copilot?

Por lo que vi a ChatGPT se le da muy bien la parte de entender el lenguaje como tal, lo cual se nota con la extensión de sus respuestas y su facilidad para entender el contexto. Por esta razón creo que ChatGPT sería una herramienta que podría usar para hacer traducciones, ya que parece que entiende lo que está diciendo en lugar de un traductor usual que busca reemplazar palabras por sus "equivalentes" en otro idioma. También podría ser una buena herramienta para resumir textos o simplificar su contenido.

Por otra parte creo que Copilot es mucho mejor para generar código; es evidente que su entrenamiento fue con código de otras personas. Sin embargo, al hacerle preguntas tuve que crear archivos separados y trucos por el estilo para aislarlo del caos de todo el archivo y poder darle a entender el contexto. Con suficientes elementos ya presentes en un archivo puede llegar a ser una herramienta que facilite las tareas que requieren de poco análisis.

Ventajas de usar AI para programar

Evidentemente la principal ventaja es la rapidez, obtienes consultas inmediatas, medianamente fundamentadas en fuentes existentes (de hecho a veces hasta parece que le hacen plagio a MDN). Como nota aparte le celebro bastante a herramientas como Bing (la cual no usé aquí por comodidad jaja) que incluyan fuentes para sus respuestas. Copilot de repente es capaz de hacer un auto-complete bastante cercano a lo que tenía en mente pero me disgusta cuando quiere hacer todo un bloque de código porque es más propenso a errores. Por otra parte ChatGPT es excelente para lo que fue creado: como un modelo de lenguaje.

Desventajas de usar AI para programar

La principal desventaja que le veo a estas herramientas es su facilidad para inventar cosas o en el caso de la programación crear funciones que no cumplen con el comportamiento esperado. También creo que hay que considerar la ética destrás de su entrenamiento ya que muchas licencias no han sido respetadas al momento de utilizar propiedad intelectual en su entrenamiento.
